Brief Introduction

 

In our solution, we further divided the single JSON file into two dataframes (Nodes, Links) using python and then join these two dataframes using a left join to get the type and country of source and target.

Main thing to focus while investigating these suspected entities is whether they are involved in IUU and the key point to focus is that if a corporation is not registered then it wont have any registered country or will have any type in the Node dataframe and if consider this hypothesis as true then we can also say that all entities (person or organization) might involve in IUU because they are dealing with IUU marked companies.

Reflection: Challenges while working with knowledge graph

 

There were several challenges while working with knowledge graph, major difficulties were as follows:

 

a.      Data quality: The data was in JSON format and could be easily integrated into the network library using tools like Observable HQ. Dataset was amalgamation of string and integer that gave a challenge to search because if a cell contains string then it wont match with an integer and vice versa to overcome this issue we have to change the datatype of each columns to a unanimous datatype and further join two dataframes (Nodes and Links) into single dataframe so that we can have a single dataframe that contains all of the significant columns.

b.     Comparederface Design: Designing interface with python was a real challenge but as compared to other technologies it was our best bet. We used streamlit library to overcome the use of CSS and HTML.

 

c.      Data handling: Some pages of the solution used data from other module of the solution and to use the modified data we had to store the dataframe into CSV to further use that dataframe again.

 

d.     Inference: initially, making sense out of the plotted data was not straight forward we have to take and change several aspects to understand the knowledge graph in depth.
